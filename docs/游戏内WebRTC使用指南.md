# 🎮 游戏内 WebRTC 使用指南

## 📍 集成位置

已为 **多人大厅** 添加了完整的 WebRTC 支持。

**文件**: `scripts/ui/multiplayer_lobby.gd`

---

## 🎯 使用流程

### 第一步：启动信令服务器

**在你的电脑上打开终端**:

```bash
# 进入 server 目录
cd d:\Games\PVZ-Godot-Dream\server

# 启动服务器
python3 webrtc_signaling_server.py
```

**预期输出**：
```
INFO:__main__:启动 WebRTC 信令服务器...
INFO:__main__:WebRTC 信令服务器运行在 ws://0.0.0.0:8080
INFO:__main__:等待客户端连接...
```

### 第二步：打开游戏

1. 在 Godot 中运行游戏
2. 进入 **主菜单** → **多人模式** / **联机** 选项
3. 看到多人大厅界面

### 第三步：选择网络模式

在大厅界面上方有多个选项卡 (Tab)：

- 🔗 **创建房间** - ENet 直连 (局域网)
- 🔗 **加入房间** - ENet 直连 (局域网)
- 🌐 **服务器中转** - 中继服务器
- 🔴 **WebRTC** ✨ **新增** - 点对点 (推荐)

---

## 🔴 WebRTC 模式详细使用

### 场景一：主机创建房间

#### 步骤 1️⃣ 选择 WebRTC 选项卡

点击大厅上方的 **"WebRTC"** 选项卡。

#### 步骤 2️⃣ 填写参数

```
信令服务器: ws://127.0.0.1:8080
    ↑ 如果和游戏在同一台电脑，用这个
    
玩家名称: 我的房间名字
    ↑ 输入你想要的名字
```

#### 步骤 3️⃣ 创建房间

点击 **"创建 WebRTC 房间"** 按钮。

#### 步骤 4️⃣ 获得房间码

几秒后，会看到：

```
✓ 房间码: [ABCD1234]
(已生成，分享给其他玩家)
```

**复制这个房间码给其他玩家**

---

### 场景二：客户端加入房间

#### 步骤 1️⃣ 选择 WebRTC 选项卡

点击大厅上方的 **"WebRTC"** 选项卡。

#### 步骤 2️⃣ 填写参数

```
信令服务器: ws://127.0.0.1:8080
    ↑ 和主机相同的地址

玩家名称: 我的名字
    ↑ 输入你的玩家名

房间码: ABCD1234
    ↑ 从主机处获得的房间码
```

#### 步骤 3️⃣ 加入房间

点击 **"加入 WebRTC 房间"** 按钮。

#### 步骤 4️⃣ 等待连接

稍等片刻，应该会看到：

```
正在加入 WebRTC 房间: ABCD1234...
```

连接成功后，会进入 **大厅界面**，看到主机和其他玩家。

---

## 🔄 连接后的操作

### 准备游戏

1. **确认玩家列表**

   大厅中显示已加入的所有玩家：
   ```
   ■ Host (主机)          未准备  [Host]
   ■ Player1              未准备
   ■ Player2              未准备
   ```

2. **点击 "准备" 按钮**

   每个玩家都需要点击 **准备** 按钮表示已准备好。

3. **选择游戏模式** (仅主机可操作)

   主机可以从下拉菜单选择：
   - 冒险模式
   - 生存模式
   - 迷你游戏
   - 解密模式
   - 自定义关卡

4. **所有玩家准备后**

   主机点击 **"开始"** 按钮启动游戏。

### 游戏中

- 所有现有游戏功能正常工作
- RPC 调用自动通过 WebRTC 点对点通信
- 网络质量会实时显示

### 游戏后

- 返回大厅继续进行下一局
- 或点击 **"离开"** 返回菜单

## 🌍 不同网络环境配置

### 本地测试 (同一台电脑)

```
信令服务器地址: ws://127.0.0.1:8080
或: ws://localhost:8080
```

### 局域网 (同一个 WiFi)

```
# 1. 查看服务器端电脑的 IP (例如 192.168.1.100)
# 2. 所有客户端使用:
信令服务器地址: ws://192.168.1.100:8080
```

**获取 IP 的方法**:

Windows:
```bash
ipconfig
# 查找 "IPv4 地址" 行
```

Mac/Linux:
```bash
ifconfig
# 查找 "inet" 地址
```

### 互联网连接 (跨地区)

```
# 1. 将服务器部署到云服务器 (阿里云/AWS/DigitalOcean等)
# 2. 所有客户端使用:
信令服务器地址: ws://your_server_ip:8080
# 或 (如果配置了域名)
信令服务器地址: ws://yourdomain.com:8080
```

## 🐛 故障排除

### 问题 1️⃣: 无法连接到信令服务器

**错误信息**:
```
正在连接 WebRTC 信令服务器: ws://127.0.0.1:8080...
[后来] 连接失败: [某个错误码]
```

**解决方案**:
1. ✅ 确保信令服务器已启动
2. ✅ 检查地址是否正确
3. ✅ 检查防火墙是否阻止了 8080 端口

### 问题 2️⃣: 房间创建成功但客户端无法加入

**解决方案**:
1. ✅ 确保信令服务器地址相同
2. ✅ 确保房间码是否输入正确
3. ✅ 等待 5-10 秒，有时信令消息需要时间

### 问题 3️⃣: 连接后看不到其他玩家

**解决方案**:
1. ✅ 等待几秒让玩家列表刷新
2. ✅ 检查网络连接
3. ✅ 尝试重新加入房间

### 问题 4️⃣: 连接中断，提示"服务器已断开"

**解决方案**:
1. ✅ 重新启动信令服务器
2. ✅ 检查网络连接
3. ✅ 尝试重新加入房间

## 📊 网络模式对比

当你在大厅中时，可以在三种模式间选择：

| 功能 | ENet 直连 | 中继服务器 | **WebRTC** |
|------|---------|---------|----------|
| 难度 | 容易 | 中等 | 中等 |
| 延迟 | 低 (局域网) | 中等 | **最低** |
| 跨域 | ❌ | ✅ | ✅ |
| 推荐 | 局域网 | 测试 | **生产** |

---

## 💡 最佳实践

### 1. 主机稳定性最重要

- 主机的网络连接需要稳定
- 如果主机掉线，所有客户端都会断开
- 建议在网络好的地方担任主机

### 2. 房间码的管理

- 房间码是 8 位字符串，只有英文大写字母和数字
- 不要截图分享，直接复制粘贴
- 房间码会在 1 小时后自动失效

### 3. 网络优化

- 使用 5GHz WiFi 而不是 2.4GHz
- 避免同时下载大文件
- 关闭其他占用网络的应用

### 4. 调试

如果遇到问题：
1. 打开 **调试控制台** (Ctrl + ~)
2. 查看是否有错误信息
3. 截图共享给开发者

## 📞 常见问题

**Q: 服务器地址写错会怎样？**
A: 会连接超时，显示"连接失败"。检查地址后重试。

**Q: 可以跳过准备直接开始吗？**
A: 不可以。必须所有人都点了准备，主机才能开始。

**Q: 中途有人掉线会怎样？**
A: 掉线的玩家会被移除，其他人继续游戏（难度不变）。

**Q: 房间码可以重复使用吗？**
A: 不可以。每个房间都有唯一的房间码。

**Q: 可以改变玩家名吗？**
A: 不可以。加入房间后玩家名是固定的。

## 🚀 进阶使用

### 自定义信令服务器地址

如果你部署了自己的信令服务器：

```
信令服务器: ws://yourserver.com:8080
```

示例 (云服务器):
```
信令服务器: ws://139.196.1.100:8080
或
信令服务器: ws://pvz.myserver.com:8080
```

### 使用安全连接 (WSS)

如果服务器配置了 SSL 证书：

```
信令服务器: wss://yourserver.com:8080
# 注意: wss 而不是 ws (Secure WebSocket)
```
