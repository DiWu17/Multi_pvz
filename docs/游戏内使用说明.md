## 🚀 三步启动

### Step 1: 启动服务器 (terminal)

```bash
cd server
python3 webrtc_signaling_server.py
```

等看到:
```
WebRTC 信令服务器运行在 ws://0.0.0.0:8080
```

### Step 2: 打开游戏

- 启动 Godot
- 运行游戏 (F5)
- 进入多人大厅

### Step 3: 创建或加入房间

**主机**:
1. 选择 WebRTC 标签页
2. 默认参数即可
3. 点击"创建 WebRTC 房间"
4. 获得房间码 (如 `ABCD1234`)
5. 分享给其他玩家

**客户端**:
1. 选择 WebRTC 标签页
2. 输入房间码
3. 点击"加入 WebRTC 房间"
4. 等待连接成功

## 🔌 实际代码修改

### 已更新的关键代码片段

**多人大厅脚本中新增**:

```gdscript
# WebRTC 相关变量
var input_webrtc_signaling_url: LineEdit = null
var input_webrtc_name: LineEdit = null
var btn_webrtc_create: Button = null
var input_webrtc_room_code: LineEdit = null
var btn_webrtc_join: Button = null

# 在 _ready 中
NetworkManager.webrtc_room_created.connect(_on_webrtc_room_created)

# 创建房间
func _on_btn_webrtc_create_pressed() -> void:
    var error = NetworkManager.create_webrtc_host(signaling_url, player_name)

# 加入房间
func _on_btn_webrtc_join_pressed() -> void:
    var error = NetworkManager.join_webrtc_room(signaling_url, room_code, player_name)

# 房间创建成功回调
func _on_webrtc_room_created(room_code: String) -> void:
    _show_lobby()  # 进入大厅
```

## 🐛 遇到问题？

### 看不到 WebRTC 选项卡

**原因**: 脚本可能没有正确附加

**解决**: 
- 确保 `multiplayer_lobby.gd` 已更新
- 重新启动 Godot

### 创建房间失败

**原因**: 信令服务器未启动或地址错误

**解决**:
- 启动服务器: `python3 webrtc_signaling_server.py`
- 检查地址拼写

### 加入房间失败

**原因**: 房间码错误或信令服务器地址不同

**解决**:
- 确认房间码是否正确
- 两端使用相同的信令服务器地址

### 连接中断

**原因**: 网络波动或主机掉线

**解决**:
- 重新启动服务器
- 检查网络连接
- 重新加入房间
