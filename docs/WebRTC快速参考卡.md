> 启动服务器 → 游戏创建房间 → 分享房间码 → 玩家加入 → 开始游戏

## 🚀 启动步骤

### 第 1 步：启动信令服务器

```bash

python3 webrtc_signaling_server.py

# 或 Mac/Linux
cd ~/Games/PVZ-Godot-Dream/server

```

**看到这个说明启动成功**:
```
WebRTC 信令服务器运行在 ws://0.0.0.0:8080
等待客户端连接...
```

### 第 2 步：打开游戏

在 Godot 中运行游戏，进入多人大厅。

---

## 🎮 游戏内使用

### 主机操作 (创建房间)

1. 选择 **WebRTC** 选项卡
2. 填写:
   - 信令服务器: `ws://127.0.0.1:8080`
   - 玩家名: 任意名字
3. 点击 **"创建 WebRTC 房间"**
4. 获得房间码 (例如: `ABCD1234`)
5. 分享给其他玩家 📲

### 客户端操作 (加入房间)

1. 选择 **WebRTC** 选项卡
2. 填写:
   - 信令服务器: `ws://127.0.0.1:8080`
   - 玩家名: 你的名字
   - 房间码: 主机给的房间码
3. 点击 **"加入 WebRTC 房间"**
4. 等待连接 ⏳

### 游戏前准备

1. 所有玩家都看到彼此时
2. 点击 **"准备"** 按钮
3. 主机选择游戏模式
4. 所有人都准备好后，主机点击 **"开始"**

## 🐛 常见问题速查

| 问题 | 原因 | 解决 |
|------|------|------|
| 连接失败 | 服务器未启动 | 启动服务器 |
| 房间创建失败 | 地址错误 | 检查信令服务器地址 |
| 无法加入 | 房间码错误 | 重新输入或重新创建 |
| 看不到玩家 | 尚未同步 | 等待几秒 |
| 中途断开 | 网络问题 | 重新连接 |

## 🔧 排查步骤

**如果无法连接**:

1. 确认服务器启动: 检查 terminal 输出
2. 确认地址正确: 复制而不是手打
3. 确认防火墙: 允许 8080 端口通过
4. 确认网络: ping 服务器地址试试
5. 查看日志: 在 Godot 调试控制台看是否有错误

```bash
# 启动服务器 (最常用)
python3 webrtc_signaling_server.py

# 后台启动 (Linux/Mac)
nohup python3 webrtc_signaling_server.py > signaling.log 2>&1 &

# 查看服务器日志
tail -f signaling.log

# 检查 8080 端口是否开放
netstat -an | grep 8080

# Windows 检查
netstat -ano | findstr 8080
```

## 💡 Pro Tips

1. **保存常用地址** - 在记事本保存信令服务器地址
2. **房间码分享** - 使用 QQ/微信/Telegram 分享，不要手打
3. **备用方案** - 如果 WebRTC 失败，切换到中继或 ENet
4. **网络稳定** - 避免同时运行下载或视频
5. **日志保存** - 出问题时保留服务器日志便于排查
