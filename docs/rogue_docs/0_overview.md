# 第0章 - 总览 (Overview)

## 核心设计理念

本方案将 PVZ 的塔防防守机制与肉鸽 (Roguelike) 的"局内构建"理念结合，参考《杀戮尖塔2》的系统深度设计。玩家通过选择植物、遗物、Buff 和应对 Debuff，构建独特的防守流派，每次游戏都有不同的策略路线。

### 设计原则

- **构建多样性**: 不同的植物、遗物和Buff组合产生不同的游戏体验
- **策略性**: 每次决策都会影响后续的可能性和风险
- **可重玩性**: 随机的敌人、事件和奖励确保每局游戏都不同
- **学习曲线**: 新手友好但有深度的策略空间

---

## 游戏流程

```
选择难度 → 获得初始卡组 → 随机事件/商店/战斗 节点选择 → 路径推进 → Boss战 → 结算
│
└─ 每5关出现Boss节点，击败所有Boss章节获胜
```

### 流程详解

1. **选择难度** - 玩家选择游戏难度（影响敌人强度和奖励规模）
2. **初始卡组** - 所有玩家获得相同的初始卡牌和资源
3. **节点选择** - 在每个关卡选择下一步的路径
4. **路径推进** - 完成选定节点（战斗/商店/事件等）
5. **重复循环** - 继续选择、完成节点
6. **Boss战** - 每5关遭遇Boss（可能有多个Boss章节）
7. **游戏结算** - 击败所有Boss后游戏结束

---

## 初始卡组 (Starting Deck)

所有玩家开局拥有相同的初始植物卡，设计为"前期可运营、但资源会逐步消耗"的平衡状态：

| 植物  | 成本 | 数量 | 说明 |
|------|------|------|------|
| **向日葵**  | 25☀ | 3 | 生产阳光的主要来源 |
| **豌豆射手**  | 100☀ | 5 | 主要伤害输出 |
| **土豆地雷**  | 25☀ | 1 | 爆发伤害 |
| **坚果墙**  | 50☀ | 1 | 防守单位 |

### 开局资源

- **开局阳光池**: 50☀
- **植物库存制**: 本局内植物卡"用一张少一张"。当某植物库存为 0 时，本局后续关卡不可再选择该植物，除非通过战后奖励、商店或事件重新获得。

### 初始资源平衡分析

- 开局50阳光能放置1个向日葵 + 1个豌豆射手，或2个坚果墙
- 向日葵有限数量意味着玩家需要通过其他方式获取阳光（遗物、Buff等）
- 豌豆射手充足量提供前期伤害输出能力
- 坚果墙仅1张，防守需要通过升级或额外获取实现

## 核心玩法循环说明

### 1. 植物选择
- **触发时机**: 每次战斗胜利后
- **机制**: 获得 3 选 1 奖励
  - 选择添加新植物到库存
  - 或选择升级已有植物

### 2. 植物升级
- **升星系统**: 通过多次选择同一植物，逐步提升其星级与属性
- **星级范围**: ★ 至 ★★★★★（5个等级）
- **属性增长**: 每升一星增加攻击力、射程或特殊能力

### 3. 遗物系统
- **获得来源**: 精英战 & Boss战掉落
- **效果**: 提供全局被动加成
- **Synergy**: 不同遗物组合产生复合效果

### 4. Buff机制
- **来源**: 场景buff、遗物buff、植物自身buff
- **组合爆炸**: 多个Buff叠加形成强大的构筑
- **策略深度**: 玩家需要考虑如何搭配Buff最大化效果

### 5. Debuff调整
- **敌方词缀**: 随机为敌人添加特殊能力
- **场地负面**: 战斗场景可能出现负面效果
- **挑战性**: 提升游戏难度和应变能力要求

## 地图生成机制

地图生成算法分为两个主要阶段：**绘制路径（画线）**和**分配房间（填色）**。

### 第一阶段：生成网格与路径（连线阶段）

游戏并不会一开始就往地图上撒各种房间，而是先搭建一个隐藏的骨架，然后在这个骨架上铺设可行的路线。

1. **基础网格**：系统首先会在后台创建一个 **7列 × 15行** 的不可见网格（代表地图的宽度为7，层数为1-15层，第16层是Boss）。
2. **生成单条路径**：
   * 系统在第1层（最底层）随机选择一个节点作为起点。
   * 接着，它会在第2层中，挑选距离该起点最近的3个节点之一（即左上、正上或右上），并在两者之间连一条线。
   * 重复这个"向上摸索连接"的过程，直到到达第15层。这就形成了一条从底到顶的完整路线。
3. **重复与叠加**：上述的单条路径生成过程会**精确地重复执行 6 次**。
4. **路径硬性规则（约束条件）**：
   * **绝对不交叉**：不同路线的连线在视觉和逻辑上都绝对不能发生交叉。
   * **确保多起点**：前两次生成的路径，其在第1层的起点绝不能是同一个。这保证了玩家开局至少有 2 个不同的入口可供选择。
   * **防止过早汇合**：为了避免所有路线在开局就合并成一条死路，系统会刻意修剪第1层和第2层之间连接到同一个节点的连线。
   * **剔除孤岛**：当6条路径全部画完后，网格中那些没有被任何线穿过的"空节点"会被直接从地图上抹去。

### 第二阶段：分配房间类型（节点填充阶段）

骨架和路线搭好后，剩下的所有节点当前都只是"空房间"，接下来系统需要决定它们具体是什么类型的房间（怪物、精英、商店等）。

#### 1. 固定层级填充
游戏首先会锁定一些特定层数的房间类型，这些是不受随机概率影响的：
* **第1层**：100% 全部是普通怪物房间。
* **第9层**：100% 全部是宝箱房间。
* **第15层**：100% 全部是篝火（休息处）。

#### 2. 计算"房间类型概率池（Bucket）"
系统会清点地图上剩下所有还没分配类型的节点总数，并严格按照设定的百分比来计算需要生成多少个特定房间：
* **问号/事件 (Event)**：约 22%
* **篝火 (Rest Site)**：约 12%
* **精英怪 (Elite)**：约 8%
* **商店 (Shop)**：约 5%

系统将这些计算出的房间数量放入一个虚拟的"抽取池"中，然后将剩余的空缺名额全部用**普通怪物房间**填满，最后将整个池子彻底打乱。

#### 3. 抽取与规则校验
系统从下到上遍历每一个无属性的节点，从"池子"中盲抽一个房间类型分配给它。为了防止出现极端恶劣的地图（比如开局连打三个精英怪），抽出的类型必须经过以下**硬性规则校验**：
* **底层保护机制**：精英怪和篝火**绝对不允许**出现在第6层之下。也就是说，地图的前5层你是绝对碰不到精英怪和篝火的。
* **禁止背靠背（防连击机制）**：精英怪、篝火、商店这三种节点**绝对不能连续出现**。如果一条路线上当前的节点是商店，与之相连的下一个节点在生成时一旦抽中商店，就会被退回重抽，保证你不会遇到连续的商店或连续的精英怪。

如果抽出的房间类型不符合上述规则，系统就会把它放回池子继续抽，直到找出一个符合规则的房间类型为止。
